## SEO implementation checklist

### Technical SEO
- [ ] All pages use `generateMeta()` from `lib/metadata.ts`
- [ ] `sitemap.ts` covers 100% of indexable URLs
- [ ] `robots.ts` explicitly allows GPTBot, ClaudeBot, PerplexityBot
- [ ] `public/llms.txt` is accessible at `/llms.txt`
- [ ] Canonical URLs set on every page
- [ ] hreflang: `en-nl` and `nl-nl` on every page pair
- [ ] `x-default` hreflang points to `/en`
- [ ] JSON-LD: Organization schema on root layout
- [ ] JSON-LD: WebPage schema on each page
- [ ] JSON-LD: Service schema on each service page
- [ ] JSON-LD: FAQPage schema on homepage + service pages
- [ ] JSON-LD: BreadcrumbList on all non-homepage pages
- [ ] JSON-LD: LocalBusiness/ProfessionalService on city pages
- [ ] HTTP security headers in `next.config.js`
- [ ] Core Web Vitals target: LCP < 2.5s, CLS < 0.1, INP < 200ms
- [ ] Images: AVIF/WebP, lazy loading, explicit width/height
- [ ] Fonts: preconnect + `display: swap`
- [ ] No render-blocking scripts
- [ ] HTTPS enforced via HSTS header

### On-page SEO
- [ ] H1 present and unique on every page
- [ ] H1 → H2 → H3 hierarchy correct (no skipped levels)
- [ ] Meta description 140–160 chars on every page
- [ ] Title tag 50–60 chars on every page
- [ ] All images have descriptive alt text
- [ ] Internal linking: every service page links to city pages
- [ ] Internal linking: every city page links to service pages
- [ ] Internal linking: city pages link to 6 other cities
- [ ] Footer contains city links (crawl entry points)
- [ ] FAQ section on homepage and service pages
- [ ] Process section with numbered steps

### Content SEO
- [ ] Each service page has 500+ words of unique content
- [ ] Each city page has 300+ words of unique content (not duplicate)
- [ ] FAQ answers are self-contained (no "as mentioned above")
- [ ] Blog/Insights: minimum 2 posts at launch, 2/month ongoing
- [ ] Every page uses keywords from `data/seo.ts` naturally
- [ ] Dutch pages use Dutch keywords (not translated EN keywords)

### GEO (Generative Engine Optimization)
- [ ] `llms.txt` is live and accessible
- [ ] `robots.ts` allows all major AI crawlers
- [ ] Homepage hero copy is factual, not marketing fluff
- [ ] FAQ answers written to be citation-ready
- [ ] Organization schema includes `knowsAbout` array
- [ ] `areaServed` on organization, service, and city schemas
- [ ] `DC.*` meta tags in `generateMeta()`
- [ ] City coverage explicitly listed in homepage body copy
- [ ] Language coverage stated explicitly (English and Dutch)
- [ ] Pricing model stated clearly (fixed-scope, no hourly)
- [ ] Response time stated clearly (one business day)

### Local/National SEO (Netherlands)
- [ ] KVK number in footer
- [ ] `@type: Country, name: Netherlands` in all schemas
- [ ] Province listed in city schemas (`containedInPlace`)
- [ ] NL-specific keywords in Dutch pages
- [ ] Dutch FAQ answers in `FAQ_NL` array
- [ ] Geo meta tags: `geo.region: NL`, `geo.country: Netherlands`
- [ ] All 20 cities have dedicated landing pages (EN + NL = 40 pages)
- [ ] City pages link to each other (internal link cluster)

After launch:

3. Submit sitemap: `https://www.veloq.nl/sitemap.xml`

5. Check Coverage report for indexing errors

## Performance budget

| Metric | Target | Threshold |
|--------|--------|-----------|
| LCP | < 1.8s | < 2.5s |
| CLS | < 0.05 | < 0.1 |
| INP | < 150ms | < 200ms |
| Total JS (gzipped) | < 100KB | < 200KB |
| Total CSS (gzipped) | < 20KB | < 50KB |
| Hero image | < 100KB WebP | — |

Animations (Framer Motion) should use `will-change: transform` and
`transform: translateY()` only — never animate `top`, `left`, `margin`,
`padding`, or `width` (triggers layout recalculation = CLS).

## Deployment

### Recommended: Vercel
- Auto-detects Next.js, zero config
- Add custom domain `veloq.nl` in Vercel dashboard
- Enable Vercel Analytics for Core Web Vitals monitoring
- Set environment variables in Vercel dashboard (not .env)

### Alternative: Netlify, Coolify, Hetzner (self-hosted)
All Next.js output modes work. For static export, set
`output: 'export'` in `next.config.js`.

## Brand name note

All files currently use "Veloq" as the brand name placeholder.
This was chosen to be:
- Short (5 chars)
- Memorable
- .nl domain likely available
- Works in both EN and NL contexts
- No obvious negative connotations in Dutch or English

When the final brand name is decided, run a global find-replace
on "Veloq" and "veloq" across all files.
