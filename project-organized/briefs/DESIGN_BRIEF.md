# Qovre - Design Brief

This file captures the current visual direction after consolidation.

## Core direction

- Primary visual reference: `project-organized/design-reference/mainlook.png`
- Do not reproduce the reference literally
- Use it for atmosphere, composition rhythm, contrast, spacing, and CTA hierarchy
- Adapt the design to the real Qovre content model

## Theme rules

- Default visual tone: premium dark
- The site should feel modern, calm, and high-trust
- Avoid generic bright SaaS styling
- Use restrained glow, strong contrast, clean typography, and deliberate spacing

## Content-led adaptation

The visual reference may show sections the real site does not currently support.
Do not fabricate portfolio or selected-project content.
If needed, translate those visual moments into real sections such as:

- featured services
- process
- trust signals
- FAQ
- city coverage
- contact CTA
- legal confidence links

## Motion rules

- Motion should support hierarchy, not dominate it
- Prefer subtle reveal, hover, and section-entry behavior
- Use existing motion dependencies and patterns already present in the active app
- Avoid decorative movement that adds noise

## Trust language

- Do not use counters based on founding year or years of experience
- If counters are used, they must rely on safe, non-time-based facts
- Prefer capability and process trust over inflated metrics

## High-level page direction

Homepage should prioritize:

1. clear value proposition
2. service clarity
3. process confidence
4. nationwide coverage language
5. strong conversion path

## Legal and policy awareness

The footer and other appropriate public surfaces should make legal assets discoverable through:

- `qovre/public/legal/`
